# AI PMO Risk Skills

AI PMO Risk Skills — набор AI-скиллов для автоматического управления проектными рисками.

Система помогает поддерживать реестр рисков в актуальном состоянии, контролировать выполнение планов митигации и формировать руководителю актуальную картину по всем проектам.

---

# Установка

## Шаг 1. Установите скиллы

Закройте Ouroboros.

Скопируйте папки со скиллами:

```text
Loop-03-Risk
SK-R01
SK-R02
SK-R03
SK-R04
```

в каталог внешних скиллов:

```text
<Ouroboros>/data/skills/external
```

---

## Шаг 2. Установите рабочую папку

Из архива скопируйте папку:

```text
AI_PMO_MVP
```

в системную папку **Documents**.

Должен получиться путь:

**Windows**

```text
C:\Users\<Имя пользователя>\Documents\AI_PMO_MVP
```

**macOS / Linux**

```text
~/Documents/AI_PMO_MVP
```

Не переименовывайте папку и не изменяйте её внутреннюю структуру.

---

## Шаг 3. Включите скиллы

В Ouroboros:

1. Откройте **Skills**.
2. Нажмите **Refresh**.
3. Выполните **Review** для всех риск-скиллов.
4. Разрешите необходимые права.
5. Включите все скиллы.
6. Перезапустите Ouroboros.

---

# Что делает система

Во время работы AI автоматически:

- проверяет новые и существующие риски;
- выявляет новые риск-сигналы;
- проверяет корректность регистрации рисков;
- оценивает вероятность и влияние;
- пересчитывает уровень риска;
- контролирует планы митигации;
- напоминает о приближающихся сроках;
- определяет необходимость эскалации;
- обновляет риск-дашборд;
- подготавливает проекты уведомлений и поручений.

---

# Входные данные

Рабочая папка:

```text
Documents/AI_PMO_MVP
```

Основные реестры находятся в папке:

```text
registry
```

Новые файлы для анализа необходимо помещать в:

```text
input
```

В качестве входных данных можно использовать:

- реестр рисков;
- статусы проектов;
- планы митигации;
- протоколы встреч;
- Excel-файлы с информацией по проектам.

---

# Запуск проверки

Для запуска полного контроля достаточно написать в обычном чате Ouroboros:

```text
Запусти полный контроль рисков.
```

или

```text
Проверь реестр рисков.
```

Пути к файлам указывать не требуется.

---

# Результаты работы

После завершения проверки обновляются рабочие реестры:

```text
Documents/AI_PMO_MVP/registry
```

в том числе:

- реестр рисков;
- журналы выполнения;
- очередь уведомлений.

Также обновляется управленческий дашборд:

```text
Documents/AI_PMO_MVP/output/dashboards
```

---

# Как работает система

```text
Входные данные
        ↓
Проверка регистрации рисков
        ↓
Поиск новых риск-сигналов
        ↓
Оценка вероятности и влияния
        ↓
Пересчёт уровня риска
        ↓
Контроль планов митигации
        ↓
Определение необходимости эскалации
        ↓
Обновление реестров
        ↓
Обновление риск-дашборда
```

---

# Что получает руководитель

После каждого запуска доступны:

- актуальный реестр рисков;
- перечень High и Critical рисков;
- список некорректно зарегистрированных рисков;
- просроченные мероприятия по митигации;
- проекты уведомлений и поручений;
- обновлённый риск-дашборд с актуальной картиной по проектам.
