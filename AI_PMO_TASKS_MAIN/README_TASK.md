# Контроль поручений AI PMO

## 1. Для чего нужен пакет

Пакет автоматизирует полный цикл работы с поручениями после встреч:

```text
протокол встречи
→ поиск реальных поручений
→ регистрация ответственного, срока и ожидаемого результата
→ контроль приближающихся и просроченных сроков
→ проверка результата по критериям приёмки
→ закрытие поручения или возврат на доработку
→ управленческий дашборд и журнал действий
```

Скилл полезен проектным менеджерам, координаторам и руководителям программ, которым нужно регулярно разбирать протоколы, контролировать выполнение поручений и не вести весь процесс вручную.

## 2. Что делает скилл

Скилл **«Контроль поручений»**:

- читает протоколы встреч в свободной форме;
- с помощью языковой модели отделяет поручения от решений, информации и открытых вопросов;
- определяет проект, ответственного, срок, ожидаемый результат и критерии приёмки;
- регистрирует поручения в локальных Excel-реестрах;
- контролирует сроки и формирует черновики напоминаний и эскалаций;
- принимает файлы с результатами исполнителей;
- проверяет результат по критериям приёмки;
- закрывает поручение либо переводит его в статус «требуется доработка»;
- обновляет виджет, Excel-дашборд и аудит действий.

**Реальные сообщения и письма пакет не отправляет.** Он формирует только записи со статусом `Draft`.

## 3. Как устроена работа

Пакет состоит из двух частей:

1. **Скилл Ouroboros** — виджет, API-маршруты, watcher и Python-runtime.
2. **Рабочая папка в Documents** — входные файлы, служебные реестры, дашборд и технический архив.

Основной сценарий:

```text
файл появляется во входной папке
→ watcher замечает новый файл
→ runtime читает его содержимое
→ создаётся отдельный внутренний AI-контекст
→ модель возвращает структурированный JSON
→ runtime проверяет JSON
→ обновляет Excel-реестры и дашборд
→ виджет показывает новые данные
```

Каждая AI-операция выполняется в новом изолированном контексте. История других запусков модели не передаётся.

## 4. Состав архива

```text
AI_PMO_TASKS_MAIN_WITH_README/
├── README.md
├── COPY_FOR_OUROBOROS/
│   └── external/
│       └── task-control-dashboard/
├── COPY_FOR_DOCUMENTS/
│   └── AI_PMO_TASKS/
└── TEST_EXAMPLES/
    ├── 01_PROTOCOL/
    │   └── COPY_TO_INPUT_MEETINGS/
    └── 02_RESULT_AFTER_PROTOCOL/
        └── COPY_TO_INPUT_RESULTS/
```

Тестовые файлы вынесены из рабочей папки, поэтому после установки они не запустятся автоматически.

# Установка

## 5. Требования

Нужно:

- установленный Ouroboros с поддержкой external skills;
- доступный Python 3 для companion-процесса;
- пакет Python `openpyxl`;
- пакет `pypdf`, только если планируется обработка PDF;
- разрешённый внутренний AI-вызов `inject_chat`;
- доступ модели для внутренних чатов диапазона `a2a`.

### Установка Python-зависимостей

macOS / Linux:

```bash
python3 -m pip install openpyxl pypdf
```

Windows:

```bash
py -m pip install openpyxl pypdf
```

Устанавливать зависимости нужно в то Python-окружение, из которого Ouroboros запускает companion-process.

## 6. Установка скила в Ouroboros

1. Полностью закройте Ouroboros.
2. Откройте в архиве:

```text
COPY_FOR_OUROBOROS/external/
```

3. Скопируйте папку:

```text
task-control-dashboard
```

4. Вставьте её в:

```text
<Ouroboros>/data/skills/external/
```

Итоговый путь должен выглядеть так:

```text
<Ouroboros>/data/skills/external/task-control-dashboard/SKILL.md
```

Не создавайте лишний уровень вложенности вида:

```text
external/COPY_FOR_OUROBOROS/external/task-control-dashboard
```

## 7. Установка рабочей папки

1. Откройте в архиве:

```text
COPY_FOR_DOCUMENTS/
```

2. Скопируйте папку:

```text
AI_PMO_TASKS
```

3. Вставьте её в системную папку `Documents` текущего пользователя.

### macOS

```text
/Users/<имя_пользователя>/Documents/AI_PMO_TASKS
```

или:

```text
~/Documents/AI_PMO_TASKS
```

### Windows

```text
C:\Users\<имя_пользователя>\Documents\AI_PMO_TASKS
```

### Linux

```text
/home/<имя_пользователя>/Documents/AI_PMO_TASKS
```

Если рабочая папка должна находиться в другом месте, задайте переменную окружения:

```text
AI_PMO_TASKS_ROOT=<полный путь к AI_PMO_TASKS>
```

## 8. Включение скила

1. Запустите Ouroboros.
2. Откройте раздел `Skills`.
3. Нажмите `Refresh`.
4. Найдите `task-control-dashboard`.
5. Выполните `Review`.
6. Разрешите:
   - `widget`;
   - `route`;
   - `companion_process`;
   - `inject_chat`.
7. Включите скил.
8. Полностью перезапустите Ouroboros.
9. Подождите 10–15 секунд.
10. Откройте вкладку **«Контроль поручений»**.

В верхней части виджета должен появиться зелёный статус **«Контур работает»**.

# Работа со скилом

## 9. Куда класть протоколы

Путь:

```text
Documents/AI_PMO_TASKS/input/meetings
```

Поддерживаемые форматы:

- `.txt`;
- `.md`;
- `.docx`;
- `.pdf` с текстовым слоем.

Скан PDF без распознаваемого текста не поддерживается: OCR в пакет не входит.

В протоколе желательно указывать:

- дату встречи;
- проект или Project ID;
- участников;
- формулировку поручения;
- ответственного;
- срок;
- ожидаемый результат;
- критерии приёмки, если они известны.

Скилл умеет работать и со свободной формой. Если критерии приёмки отсутствуют, модель сформирует измеримые критерии на основе ожидаемого результата.

### Как запустить обработку протокола

Есть два способа:

1. Скопировать файл в `input/meetings` и подождать автоматического запуска.
2. Открыть виджет → вкладка **«Действия»** → нажать **«Обработать протоколы»**.

Перед автоматическим чтением watcher ждёт 15 секунд, чтобы не начать обработку во время копирования файла.

## 10. Куда класть результаты исполнителей

Путь:

```text
Documents/AI_PMO_TASKS/input/results
```

Поддерживаемые форматы:

- `.txt`;
- `.md`;
- `.docx`;
- `.pdf` с текстовым слоем.

Имя файла должно начинаться с ID поручения:

```text
TASK-20260715-001_v1.txt
TASK-20260715-001_v2.docx
```

Где:

- `TASK-20260715-001` — фактический `task_id` из виджета или реестра;
- `v1`, `v2` — версия результата.

### Как запустить проверку результата

1. Скопируйте файл в `input/results`.
2. Откройте виджет → **«Действия»**.
3. Нажмите **«Проверить результаты»**.

Система сопоставит файл с поручением, проверит обязательные критерии и выставит один из итогов:

- `Accepted` → поручение закрывается;
- `Rework_Required` → поручение возвращается на доработку.

## 11. Контроль сроков

Во вкладке **«Действия»** нажмите:

```text
Проверить сроки
```

Система пересчитает:

- активные поручения;
- срок приближается;
- срок сегодня;
- просрочено;
- уровень эскалации;
- Draft-напоминания и Draft-эскалации.

Для проверки сроков языковая модель не используется.

## 12. Полный контроль

Кнопка **«Запустить полный контроль»** выполняет последовательно:

```text
разбор новых протоколов
→ регистрация поручений
→ проверка новых результатов
→ контроль сроков
→ обновление дашборда
```

## 13. Автоматический запуск

По умолчанию включён профиль `HYBRID`:

- команды из виджета — сразу;
- новые входные файлы — автоматически;
- полный страховочный проход — каждые 180 минут;
- проверка папок watcher — каждые 2 секунды;
- задержка перед чтением нового файла — 15 секунд;
- часовой пояс — `Europe/Moscow`.

Плановые проходы привязаны к трёхчасовым границам московского времени: 00:00, 03:00, 06:00 и далее.

Настройки находятся в служебном файле:

```text
registry/10_loop_trigger_settings.xlsx
```

Менять их нужно только при понимании логики runtime.

# Где смотреть результат

## 14. Виджет Ouroboros

Виджет содержит пять вкладок:

- **Обзор** — ключевые показатели и ближайшие проблемы;
- **Поручения** — список зарегистрированных поручений;
- **Действия** — ручные кнопки запуска;
- **Журнал** — история действий и запусков;
- **Настройки** — состояние watcher, пути и расписание.

## 15. Excel-дашборд

Основной пользовательский файл:

```text
Documents/AI_PMO_TASKS/output/dashboards/10_task_dashboard.xlsx
```

В нём отображаются:

- общее количество поручений;
- активные;
- приближающиеся сроки;
- просроченные;
- результаты на проверке;
- поручения на доработке;
- закрытые поручения;
- критичные поручения;
- последние действия агента;
- показатели по проектам.

## 16. Служебные реестры

Папка:

```text
Documents/AI_PMO_TASKS/registry
```

содержит внутренние Excel-файлы runtime:

| Файл | Назначение |
|---|---|
| `01_projects.xlsx` | справочник проектов |
| `02_people.xlsx` | сотрудники, псевдонимы и роли эскалации |
| `03_task_register.xlsx` | поручения, критерии, результаты и история |
| `04_task_rules.xlsx` | правила регистрации, сроков и приёмки |
| `05_message_outbox.xlsx` | Draft-напоминания и эскалации |
| `06_task_event_queue.xlsx` | очередь событий |
| `07_task_control_state.xlsx` | состояние и история запусков |
| `08_skill_action_log.xlsx` | аудит действий |
| `09_source_register.xlsx` | обработанные протоколы и результаты |
| `10_loop_trigger_settings.xlsx` | настройки watcher и расписания |

Эти файлы не являются пользовательским входом. Не удаляйте и не переименовывайте их.

По умолчанию исходные протоколы и результаты остаются во входных папках для аудита. Повторная обработка предотвращается по SHA-256 хэшу файла.

# Проверка после установки

## 17. Тест 1 — разбор протокола

1. Возьмите файл:

```text
TEST_EXAMPLES/01_PROTOCOL/COPY_TO_INPUT_MEETINGS/TEST_01_protocol_AI_PMO.txt
```

2. Скопируйте его в:

```text
Documents/AI_PMO_TASKS/input/meetings
```

3. Откройте виджет.
4. Нажмите **«Обработать протоколы»** либо подождите автоматического запуска.
5. Дождитесь завершения.

Ожидаемый результат:

- обработан один протокол;
- зарегистрированы три поручения;
- поручения появились во вкладке **«Поручения»**;
- дашборд обновился;
- в журнале появились этапы разбора и регистрации.

На этом этапе файл результата копировать не нужно.

## 18. Тест 2 — проверка результата

Тест выполняется только после успешного теста протокола.

1. Откройте виджет и найдите фактический ID первого поручения.
2. Возьмите файл:

```text
TEST_EXAMPLES/02_RESULT_AFTER_PROTOCOL/COPY_TO_INPUT_RESULTS/TASK-20260715-001_v1.txt
```

3. Если фактический ID отличается, переименуйте файл так, чтобы имя начиналось с нужного `task_id`.
4. Скопируйте файл в:

```text
Documents/AI_PMO_TASKS/input/results
```

5. Нажмите **«Проверить результаты»**.
6. Проверьте итог во вкладках **«Поручения»** и **«Журнал»**.

# Обновление и эксплуатация

## 19. Обновление установленной версии

При обновлении существующей установки:

1. Сделайте резервную копию:

```text
Documents/AI_PMO_TASKS
```

2. Закройте Ouroboros.
3. Замените только папку скила:

```text
<Ouroboros>/data/skills/external/task-control-dashboard
```

4. Не заменяйте рабочую папку `AI_PMO_TASKS` шаблонной версией, иначе можно потерять накопленные поручения и журналы.
5. Выполните `Refresh`, `Review`, `Enable` и перезапустите Ouroboros.

Для полностью чистого теста можно заменить и рабочую папку, предварительно сохранив старую.

## 20. Быстрая диагностика

В папке скила есть файл:

```text
diagnose.py
```

Его можно запустить из терминала в том же окружении, что и Ouroboros:

```bash
python3 diagnose.py
```

Он проверяет:

- найден ли workspace;
- доступны ли `HOST_SERVICE_URL` и `HOST_SERVICE_TOKEN`;
- существует ли heartbeat;
- состояние локального runtime.

## 21. Частые проблемы

### Виджет не появился

- проверьте правильность пути к папке скила;
- выполните `Skills → Refresh`;
- выполните `Review` и `Enable`;
- полностью перезапустите Ouroboros.

### Watcher остановлен или heartbeat устарел

- проверьте разрешение `companion_process`;
- перезапустите Ouroboros;
- убедитесь, что Python 3 доступен companion-процессу;
- откройте вкладку **«Настройки»** и проверьте путь watcher;
- запустите `diagnose.py`.

### Host Service недоступен

Проверьте:

- разрешение `inject_chat`;
- наличие `HOST_SERVICE_URL` и `HOST_SERVICE_TOKEN` у companion-процесса;
- полный перезапуск Ouroboros после Review.

### Ошибка `403 Access denied by security policy`

Это не ошибка протокола или JSON. Внутренний AI-канал дошёл до провайдера, но провайдер отклонил авторизацию.

Нужно проверить доступ к модели для:

```text
internal chat range: a2a
permission: inject_chat
```

Работа обычного чата не гарантирует доступ companion-процесса к внутреннему AI-каналу.

### PDF не читается

- установите `pypdf`;
- убедитесь, что PDF содержит текстовый слой;
- для скана используйте предварительное OCR или сохраните текст в TXT/DOCX.

### Виджет не обновляет данные

- нажмите кнопку обновления в правом верхнем углу виджета;
- проверьте вкладку **«Журнал»**;
- убедитесь, что файл не начинается с `README`, `.`, `00_` или `КЛАСТЬ_` — такие файлы игнорируются;
- убедитесь, что файл имеет расширение TXT, MD, DOCX или PDF;
- подождите минимум 15 секунд после копирования файла.
